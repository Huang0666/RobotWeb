# RobotWeb 决策记录

本文件记录影响长期实施的产品和技术决策。

## 2026-05-07 - 本地记忆作为执行权威来源

- 决策：后续自动执行不得只依赖聊天上下文，必须以 `docs/execution-memory/` 为恢复入口。
- 原因：10 天执行周期较长，心跳、上下文压缩、换线程都可能导致聊天记忆不完整。
- 影响：每个小任务完成后必须更新本地文件，再推进下一项。

## 2026-05-07 - 坐标系统以 z 为高度

- 决策：产品和机器人业务层统一使用 `x/y` 表示平面，`z` 表示高度。
- 原因：符合机器人、点云、BIM/IFC 和工程用户直觉。
- 影响：Three.js 的 `y-up` 只允许存在于渲染适配层。

## 2026-05-07 - 点击点位不自动移动

- 决策：点击地图或巡检点只允许选中目标，不直接控制机器人移动。
- 原因：真实机器人控制需要明确确认，避免误操作。
- 影响：必须点击“前往目标”或运行任务后，机器人才能执行。

## 2026-05-07 - Web 不替代机器人本体避障

- 决策：Web 负责地图、点位、任务、展示和高层控制；真实避障由机器人本体负责。
- 原因：机器人本体具备实时传感和安全控制能力，Web 做高层任务更稳妥。
- 影响：Web 路线用于任务规划和展示，不承诺局部避障。
